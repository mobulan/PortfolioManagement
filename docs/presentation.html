主要风险

技术风险

  • `page.jsx` 仍较单体。
  • localStorage 是主数据库，迁移必须保守。
  • ESM warning 会影响验证输出清晰度。

产品风险

  • 导入冲突解释不足会降低信任。
  • 再平衡草案需要更强解释性。
  • 乱码文案影响汇报和使用体验。

质量风险

  • 浏览器级测试未闭环。
  • Supabase 同步未实测。
  • 复杂 UI 流程仍需手工验收。

汇报结尾

当前项目已经进入“核心闭环可用，质量和体验待收口”的阶段。

下一步不建议立刻开大功能，而应先完成 P0 收口：浏览器级回归、导入/交易提示、乱码修复和同步验证。之后再进入 dashboard 可视化、Excel 迁移和历史 NAV 回测。
